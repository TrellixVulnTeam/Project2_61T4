                                                                                                                    Vitor Freitas Mod Vladimir Donets • 2 years ago

                                                                                                                    It is definitely something to pay attention. It will really depends on how effective is the queryset, if the export model has foreign keys (so you need to retrieve the related properties using select_related) and how many fields are you exporting.

                                                                                                                    Some tests locally with the auth.User model, a postgresql database with 5000 users -- exporting the whole model took a solid 15 seconds (because it exports also user_groups and user_permissions)

                                                                                                                    Another test with the same database, but now exporting no related fields, it runs in less than 1 second.

                                                                                                                    Another point of attention is with date/datetime fields. It gets kinda slow parsing them.

                                                                                                                    So for datasets < 20k~50k rows with not very complex querysets I think it should work fine.

                                                                                                                    But for really big datasets I would recommend using asynchronous tasks, using Celery for example
